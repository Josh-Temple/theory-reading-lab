# Default Workflow

## 1) Define theme scope
- Clarify the theme question, boundaries, and expected audience level.
- Record scope decisions in `themes/<theme>/overview.md`.
- Keep one clear center of gravity per planned episode.

## 2) Collect sources
- Add reusable books, primary texts, and secondary literature to `sources/bibliography.md`.
- Add web materials with annotations to `sources/url-index.md`.
- Prefer high-signal sources over large undifferentiated lists.

## 3) Preserve reusable research notes
- Distill concepts, arguments, tensions, and terminology in `themes/<theme>/research-notes.md`.
- Keep source-based claims, interpretation, and teaching simplifications distinguishable where useful.
- Preserve notes that can support multiple episodes.

## 4) Plan episode order
- Sequence episodes from foundational concepts to debate/critique.
- Keep the plan concise and update theme overviews as needed.

## 5) Draft scripts
- Draft script files (e.g., `episode-01.md`) from research notes.
- Keep scripts audience-facing, teachable, and conceptually precise.
- Do not paste research-note fragments directly into scripts.

## 6) Revise scripts
- Verify conceptual precision against sources.
- Tighten transitions, remove redundancy, and improve spoken clarity.
- Maintain calm, explanatory prose instead of inflated rhetoric.

## 7) Optional TTS adaptation
- Prepare an alternate read-optimized pass if audio production requires it.
- Prefer shorter sentence units and explicit transitions for listening comprehension.

## Preserve vs. do not preserve
### Preserve
- Stable bibliographic references
- Reusable conceptual notes and distinctions
- Final or near-final script drafts
- Lightweight decision records that affect future work

### Do not preserve
- Temporary scratch text with no reuse value
- Unchecked paraphrases with unknown source traceability
- Repetitive brainstorming dumps that obscure core notes
- Raw AI output saved without curation
